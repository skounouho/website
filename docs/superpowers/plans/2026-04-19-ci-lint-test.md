# CI Lint/Test Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a GitHub Actions workflow that runs lint, typecheck, tests, and `next build` on every PR to `main` (and every push to `main`), with automatic `eslint --fix` commits on PR branches. Separately, enable Vercel Agent for AI PR review.

**Architecture:** One workflow file, one job, one Node version. Auto-fix runs inside the same job as validation so CI always sees the post-fix state within a single run. Vercel Agent is a dashboard toggle — no repo changes. Branch protection (required status checks) is deliberately deferred.

**Tech Stack:** GitHub Actions, Node.js 24 LTS, existing scripts (eslint, vitest, next build), TypeScript 5 (`tsc --noEmit` via a new `typecheck` npm script), `stefanzweifel/git-auto-commit-action@v5`, Vercel Agent (public beta).

**Testing note:** CI workflow files can't be unit-tested meaningfully — the real verification is opening a PR and watching the run. This plan's "test" steps validate syntax locally (YAML parse, `tsc --noEmit`, `eslint`) and defer behavioral verification to Task 4.

## Task 1: Add `typecheck` script to package.json

**Files:**
- Modify: `package.json`

- [ ] **Step 1: Add the script**

Edit `package.json` to add a `typecheck` entry in the `scripts` block. The final scripts block should be:

```json
"scripts": {
  "dev": "next dev -H 0.0.0.0",
  "build": "next build",
  "start": "next start",
  "lint": "eslint",
  "typecheck": "tsc --noEmit",
  "test": "vitest run",
  "test:watch": "vitest",
  "test:ui": "vitest --ui"
},
```

- [ ] **Step 2: Verify the script runs cleanly**

Run: `npm run typecheck`
Expected: exits 0 with no output (or a single-line banner from `tsc`). If any `.ts`/`.tsx` file has a type error, fix the test before proceeding — a broken baseline will make Task 4's PR run red.

- [ ] **Step 3: Commit**

```bash
git add package.json
git commit -m "chore: add typecheck script"
```

## Task 2: Create the CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the directory**

```bash
mkdir -p .github/workflows
```

- [ ] **Step 2: Write the workflow file**

Create `.github/workflows/ci.yml` with exactly this content:

```yaml
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

concurrency:
  group: ci-${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: write
  pull-requests: read

jobs:
  verify:
    name: Lint, typecheck, test, build
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          ref: ${{ github.head_ref || github.ref }}
          fetch-depth: 0

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: '24'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Auto-fix with eslint
        id: autofix
        if: >-
          github.event_name == 'pull_request' &&
          github.event.pull_request.head.repo.full_name == github.repository &&
          github.actor != 'github-actions[bot]' &&
          !startsWith(github.event.head_commit.message, 'chore: apply eslint --fix')
        run: npx eslint . --fix

      - name: Commit auto-fix
        if: >-
          steps.autofix.outcome == 'success' &&
          github.event_name == 'pull_request' &&
          github.event.pull_request.head.repo.full_name == github.repository
        uses: stefanzweifel/git-auto-commit-action@v5
        with:
          commit_message: 'chore: apply eslint --fix'

      - name: Lint
        run: npm run lint

      - name: Typecheck
        run: npm run typecheck

      - name: Test
        run: npm run test

      - name: Build
        run: npm run build
```

- [ ] **Step 3: Validate YAML syntax locally**

Run: `node -e "require('yaml').parse(require('fs').readFileSync('.github/workflows/ci.yml', 'utf8'))" && echo OK`
Expected: prints `OK`. If this fails, fix the YAML before committing — a malformed workflow won't even show up in the GitHub Actions UI.

(The `yaml` package is already a dependency of this repo, so no install is needed.)

- [ ] **Step 4: Sanity-check the scripts it references exist**

Run: `npm run | grep -E '^  (lint|typecheck|test|build)$'`
Expected: all four lines present. If `typecheck` is missing, Task 1 wasn't completed — go back and finish it.

- [ ] **Step 5: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add lint/typecheck/test/build workflow for PRs to main"
```

## Task 4: Open PR and verify behavior end-to-end

**Files:**
- (No file changes.)

- [ ] **Step 1: Push the branch**

Run: `git push -u origin chore/ci-lint-test`
Expected: branch pushed; a hint from GitHub to open a PR.

- [ ] **Step 2: Open the PR**

Run:
```bash
gh pr create --base main --title "ci: add lint/typecheck/test/build workflow + typecheck script" --body "$(cat <<'EOF'
## Summary

- Add `npm run typecheck` script (`tsc --noEmit`).
- Add `.github/workflows/ci.yml` running lint → typecheck → test → build on PRs to `main` and pushes to `main`.
- Auto-fix via `eslint --fix` commits back to the PR branch (same-repo PRs only).

## Test plan

- [x] Vercel Agent enabled in dashboard (one-time setup)
- [ ] CI run is green on this PR (baseline — no code changes other than the workflow + typecheck script)
- [ ] Vercel Agent posts a review comment on this PR
- [ ] Follow-up: open a throwaway PR that introduces a fixable lint issue; confirm auto-fix commit appears and subsequent steps pass
- [ ] Follow-up: open a throwaway PR that introduces a type error; confirm `typecheck` step fails
EOF
)"
```

Expected: `gh` prints the new PR URL.

- [ ] **Step 3: Watch the run**

Run: `gh pr checks --watch`
Expected: the `CI / Lint, typecheck, test, build` check transitions to success. If it fails, inspect with `gh run view --log-failed` and fix — do not merge a red PR.

- [ ] **Step 4: Confirm Vercel Agent posted**

Run: `gh pr view --comments` (or check the PR in the browser).
Expected: at least one Vercel Agent comment is present. If absent after ~5 minutes, re-check the dashboard toggle from Task 3.

- [ ] **Step 5: Merge**

Once green and the Agent comment is present:
```bash
gh pr merge --squash --delete-branch
```

- [ ] **Step 6: Confirm main stays green post-merge**

Run: `gh run list --branch main --limit 1`
Expected: the most recent run on `main` is `completed / success`. This validates the `push` trigger half of the workflow.

- [ ] **Step 7: Follow-up verification (optional, recommended)**

Open a throwaway branch with a trivial fixable lint violation (e.g. an unused import in a file under `app/`). Open a PR. Confirm:

1. The `Commit auto-fix` step produces a commit on the PR branch with message `chore: apply eslint --fix`.
2. The subsequent `Lint` / `Typecheck` / `Test` / `Build` steps all pass.

Close the throwaway PR without merging.

Then open a second throwaway with a type error (e.g. `const x: number = "hello"` somewhere). Confirm the `Typecheck` step fails and the overall check is red. Close without merging.

If both follow-ups behave as expected, the workflow is validated.

## Rollback

If the workflow misbehaves on `main` after merge (e.g. every push runs red for reasons unrelated to the code), delete the workflow file and push:

```bash
git checkout main && git pull
git rm .github/workflows/ci.yml
git commit -m "revert: remove broken CI workflow"
git push
```

The `typecheck` script addition is harmless standalone and can stay.
